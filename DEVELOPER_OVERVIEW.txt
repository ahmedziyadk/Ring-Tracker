Ring Tracker - Developer Overview
=================================

Purpose
-------
Ring Tracker is a Flutter + Firebase app for managing gemstone/ring orders.
It is used by Admin, Office staff, and Ring Makers to create orders, assign
work, track progress, share order links, and manage delivery/status updates.


Technology Stack
----------------
- Flutter app with Android and web support.
- Firebase Authentication for login.
- Cloud Firestore for users, orders, notifications, and counters.
- Firebase Storage for order/model images.
- Provider for authentication state.
- PDF/printing packages for order reports.
- url_launcher for WhatsApp links.
- flutter_contacts for saving customer contacts.


Main App Flow
-------------
App entry point:
- lib/main.dart

Startup:
1. Firebase is initialized.
2. AuthProvider listens to FirebaseAuth auth state.
3. If a public order link is opened with orderId and token query parameters,
   the app opens the public order view.
4. Otherwise, the signed-in user's Firestore role controls which dashboard opens:
   - role = admin  -> AdminDashboard
   - role = office -> OfficeDashboard
   - role = maker  -> MakerDashboard


User Roles
----------
Users are stored in Firestore collection:
- users/{uid}

Important user fields:
- role: admin, office, or maker
- name: display name
- email: login email
- mobileNumber: maker WhatsApp number, used for sending order details to maker

Admin:
- Can view all orders.
- Can add/edit/delete orders.
- Can manage makers.
- Can view maker-wise orders.
- Can export PDFs.
- Can resend customer WhatsApp links.
- Can send order details to maker via WhatsApp.

Office:
- Can add new orders.
- Can view only orders created by that Office user.
- Office-created orders still appear in Admin.
- Has maker side menu like Admin for viewing maker-wise orders.
- Office order customer names are generated as AZ01, AZ02, AZ03, etc.

Maker:
- Can view orders assigned to their Firebase user ID.
- Can update status.
- Can add maker remarks.
- Can view uploaded model/reference images.


Firestore Collections
---------------------
orders:
Stores all order data.

Important order fields:
- orderNumber: generated order number like ORD-1234
- customerToken: random token for secure public order links
- customerName
- mobileNumber
- makingType
- stoneType
- size
- ringMakerId: maker user UID
- ringMakerName
- totalAmount
- advancedAmount
- modelImageUrl
- imageUrls
- noteToMaker
- isUrgent
- orderDate
- expectedDelivery
- status: pending, in_progress, rework, completed
- makerRemark
- createdBy: UID of admin/office user who created the order
- createdByRole: admin or office

users:
Stores app users and makers.

notifications:
Stores maker remark notifications for Admin.

counters:
Used for office customer name numbering.
- counters/officeCustomerName
- next: next AZ sequence number

Order Creation
--------------
Order creation happens mainly in AddOrderScreen.

Admin order:
- Admin enters normal customer name and full order details.
- Order is saved with createdBy as the Admin UID.
- createdByRole is saved as admin.

Office order:
- AddOrderScreen is opened with isOfficeOrder = true.
- Customer name is auto-filled with the next AZ number.
- Example sequence: AZ01, AZ02, AZ03.
- The generated name can be tapped and edited manually before saving.
- Order is saved with createdBy as the Office UID.
- createdByRole is saved as office.

AZ customer names:
- Preview is generated by scanning existing AZ names.
- Final reservation uses counters/officeCustomerName transaction to avoid duplicates.


Order Statuses
--------------
Supported status values:
- pending
- in_progress
- rework
- completed

Meaning:
- pending: order is created but not actively being worked on.
- in_progress: maker has started the order.
- rework: returned/rework/correction needed.
- completed: order is finished.

Rework behavior:
- Admin dashboard has a Return / Rework section.
- Maker dashboard has a Return / Rework section.
- Maker and Admin can set status to Rework.
- Rework is treated as not completed.


WhatsApp Features
-----------------
Customer WhatsApp link:
- Created after adding an order when customer mobile number exists.
- Can be resent from OrderDetailScreen.
- Link format:
  https://gemstone-orders.web.app/?orderId={orderId}&token={customerToken}

Customer public link shows:
- Customer name
- Customer mobile
- Order details
- Payment details
- Model images

Maker WhatsApp:
- Available from OrderDetailScreen.
- Uses maker mobileNumber saved in users/{makerUid}.
- If maker mobile number is missing, app shows a warning.
- Message includes:
  - Customer name
  - Making
  - Size
  - Order date
  - Expected delivery shown as Malayalam label: സമയ പരിധി
  - Ring maker remark
  - Maker-safe public order link

Maker-safe link format:
  https://gemstone-orders.web.app/?orderId={orderId}&token={customerToken}&view=maker

Maker-safe public link hides:
- Customer phone number
- Payment details


Contact Saving
--------------
AddOrderScreen can save customer contact to the phone.

Behavior:
- Requests contacts permission.
- If multiple contact accounts/locations exist, user can select where to save.
- Saves customer name and mobile number.
- Office-generated AZ names must be manually editable before saving contact.


PDF Features
------------
PDFs are generated from OrderDetailScreen and AdminDashboard.

OrderDetailScreen:
- Full Order Sheet includes payment details.
- Ring Maker Sheet hides payment details.

AdminDashboard:
- Can export all orders PDF.


Admin Dashboard
---------------
AdminDashboard shows all orders from Firestore.

Sections:
- Urgent Orders
- Pending
- Due Soon
- Return / Rework
- Completed
- Overdue

Admin drawer:
- Lists ring makers.
- Shows maker order counts.
- Opens maker-wise orders screen.
- Includes Manage Makers.


Office Dashboard
----------------
OfficeDashboard shows only orders created by the signed-in Office user.

It includes:
- Office order list.
- New Order button.
- Sidebar maker list like Admin.
- Maker-wise order navigation.

Admin still sees Office-created orders because Admin reads all orders.


Maker Dashboard
---------------
MakerDashboard shows only orders where:
- order.ringMakerId == signed-in maker UID

Maker can:
- View order details.
- View uploaded model images.
- Add/update maker remark.
- Update status: Pending, In Progress, Rework, Completed.


Public Order View
-----------------
The public view is opened from a URL containing:
- orderId
- token

Security:
- The screen loads the order by orderId.
- It checks that the token matches customerToken in Firestore.
- If token does not match, the link is rejected.

Modes:
- Customer mode: default.
- Maker mode: enabled with view=maker.

Maker mode hides payment and customer phone.


Maker Management
----------------
Admin can add and edit makers in ManageMakersScreen.

Adding a maker:
- Creates Firebase Authentication user.
- Creates Firestore users/{uid} document.
- Sets role = maker.
- Saves name, email, and mobileNumber.

Important:
- Adding a maker signs out the current admin due to FirebaseAuth behavior.
- The app currently tells the user to log in again after maker creation.


Known Notes for Future Developers
---------------------------------
- Some files may contain older commented-out code from previous versions.
  Prefer the active implementation after the real import section.
- Several UI files use withOpacity, which newer Flutter versions mark
  deprecated in analyzer output. This is warning-level only.
- Some rupee symbols and emoji were previously corrupted by encoding issues.
  Newer WhatsApp messages intentionally use plain text where possible.
- The customer public link token is not a full authentication mechanism,
  but it prevents casual access without the token.
- Firestore security rules should be reviewed before production deployment.


Common Setup Steps
------------------
Creating an Office user:
1. Create the user in Firebase Authentication.
2. Copy the Firebase Auth UID.
3. Create Firestore document users/{uid}.
4. Add:
   role: office
   name: Office
   email: office email

Creating a Maker user:
- Prefer using Manage Makers in the app.
- Add WhatsApp number so maker WhatsApp button works.
